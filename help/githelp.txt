Как использовать GitHub

1. Создайте учетную запись на GitHub.
можно заполнить профиль информацией о своих навыках и проектах и дать ссылку на профиль в гитхаб в своем резюме.

2. Рабочий депозиторий по проекту https://github.com/avladova/lms-analytics.
подайте запрос на участие - пришлите @avladova свой ник на гитхаб

3. Клонировать репозиторий на свой локальный компьютер не обязательно на первом этапе.

4. Работа с кодом:
Осуществляйте изменения в коде, сохраняйте их и отправляйте на GitHub.
можно работать в нашем проекте posit.cloud: https://posit.cloud/spaces/660761/join?access_code=xMRIfhyF8EEkGa_mi9W5SVVHn9HzrO5z1gkTypdg  
сохраняя файлы в development/R

5. Ветка (Branch):
Создавайте ветки для работы над разными частями проекта.
позже обсудим

6. Pull-запросы:
Отправьте pull-запрос для принятия изменений в основной ветку.
позже обсудим

7. Слияние (Merge):
Сливайте ветки, когда они готовы к объединению.
позже обсудим

8. Проекты на GitHub 
для управления проектом LMS dashboard создана доска с 4 состояниями задач: Backlog, Ready, In progress, In review. 
задачи будут назначаться, переводиться на следующий уровень, решаться. по выполненным задачам производится оплата.

9. Issues & Discussions
Вопросы ориентированы на выполнение задач, используются для отслеживания ошибок, запросов функций и других рабочих элементов, в то время как обсуждения более открыты, подходят для мозгового штурма, общих вопросов и отзывов сообщества.

8. Сообщество:
Общайтесь с другими разработчиками на GitHub и получайте поддержку.
